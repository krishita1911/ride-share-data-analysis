# Cyclistic Ride Share Data Analysis Case Study

## Objective
Analyze behavioral differences between casual and annual users using 12 months of ride-share data and derive actionable business insights.

## Dataset
This case study uses ride-share trip data provided as part of a Google Analytics course.
Cyclistic is a fictional company and the dataset is used for educational and analytical practice.

## Tools Used
- SQL (data cleaning, querying, aggregation)
- Google Sheets (pivot tables and data visualizations)

## Process
- Cleaned and merged monthly datasets using SQL
- Performed aggregations and trend analysis using SQL queries
- Exported results to Google Sheets
- Created pivot tables and charts for visualization
- Interpreted insights with a business perspective

## Key Insights 

### Weekly Patterns
- Annual members showed consistent ride duration across weekdays
- Casual users had longer rides during weekends
- Clear behavioral differences based on user type
  
### Monthly Trends
- Ride volume increased during summer months
- Casual rider usage peaked during holiday periods
- Annual members showed stable usage across months
  
## Recommendations
- Target casual users with weekend promotions
- Improve retention strategies for weekday commuters (primarily annual members)
- Optimize bike placement near leisure hotspots

## Analysis Structure
- `/data` → dataset notes
- `/sql` → SQL queries used in analysis
- `/sheets` → Google Sheets outputs (pivot tables & charts)
- `/visuals` → final visualizations
